Getting started

The artifact consists of i) a Front End: Forward Verifier; and ii) a Back End: the term rewriting system.
There are two ways to evaluate the artifact:

  • by using a virtual machine (VM) provided;
  • by using your local machine with OCaml version 4.10.0.
Below, we provide guidance for both approaches.

Using the Virtual Machine

You need to get a VirtualBox (tested w/ version 6.1.26) on your system and import the VM TempAS.ova. Please, note that:

  • Username and password are both tacas2022.

The source code can be found in the folder /home/tacas2022/Semantics_HIPHOP.
To compile it, you need to run cd /home/tacas2022/Semantics_HIPHOP && dune build in a terminal.

    Checking Reusability of the artifact

    Description of the project

    The project needs to be built together with a submodule sleek, which is included in the main Semantics_HIPHOP repository.
    The sleek submodule implements the back-end, supporting the front-end developed in Semantics_HIPHOP.

    Tool structure

    Most of the source is boiler plate to tie together parsers, the interesting files are:

    • ast.ml contains the AST for parsing HipHop.js source file.
    • sleek/ast.mli contains the AST for parsing Timed Effects Specification.
    • sleek/checker.ml contains the interface to Z3

    • hip.ml is the main entrance of the front-end.
    • trs.ml is the main entrance of the back-end.
